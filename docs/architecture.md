# architecture.md — 입력 → 해석 → 전망 → 보고초안

> `PRD.md` 기준의 MVP 동작 구조 요약. 첫 버전은 `월 단위 조회·설명`과 `전사 총계 전망 작성 지원`에 집중한다.

| 섹션 | 내용 |
|---|---|
| Goal | 채널 물량 관련 질문에 대해 `조회·설명 + 전사 총계 전망 + 채널별 방향성 + 보고용 초안`을 제공한다 |
| Input | 사용자 질문(텍스트) · 월 단위 업적 물량 집계 데이터 · 조직 효율 숫자 · 시책 세부내용 요약 · 신상품 정보 · 영업일수 · CSM · 시장 규모 · 채널별 특이사항 · 자유 메모 3~5줄 · 회의자료 업로드 후 요약 |
| Input Rule | 첫 버전은 비식별 또는 집계 데이터 우선 · 채널 이슈 입력 최소 항목은 `이슈 내용 / 적용 채널 / 기간 / 확신도` · 개인 식별정보와 상세 민감정보는 제외 |
| Process | ① 질의 유형 분류(조회형/설명형/예측형) → ② 관련 수치·이슈·메모 검색 → ③ 최신성·누락 여부 점검 → ④ 전사 총계 조회값 또는 전망 범위 산출 → ⑤ 채널별 방향성 생성 → ⑥ 상위 3개 영향 요인과 흔들림 조건 정리 → ⑦ `markdown` 기준 보고용 문장 초안 구성 |
| Logic | 출력 순서는 `전사 총계 먼저 → 채널별 방향성 → 상위 3개 영향 요인 → 흔들림 조건 → 보고 초안`으로 설계한다 |
| Tools | Supabase(월 단위 집계 데이터 저장·조회) · BizRouter LLM(설명·전망·보고 문장 생성) · (선택)질의 분류기(조회형/설명형/예측형 라우팅) · 엑셀/회의자료 업로드 파이프라인 |
| Output | 전사 총계 전망 범위와 중심값 · 채널별 증가/보합/감소 방향성 · 상위 3개 영향 요인 · 전망이 상단/하단으로 흔들릴 수 있는 조건 · 보고용 서술 초안 |
| Output Format | MVP 기본 형식은 `markdown` · 후속 확장 대상은 `json/csv/report` |
| HITL | 데이터 최신성은 운영 담당자가 확인 · 총계 전망과 채널별 방향성은 실무자가 검토 · 경영진 제출용 문안은 의사결정자 또는 보고 책임자가 최종 승인 |
| Failure | 데이터 최신화 누락 시 전망 중단 또는 경고 · 근거 부족 시 단정 금지 및 `추가 확인 필요` 표시 · 변수 충돌 시 복수 가능성 또는 흔들림 조건으로 제시 |
| Out of Scope | 주별/일별 실적 분석 · 임원 보고 직전 사전 점검 로직 · `json/csv/report` 본격 지원 · 채널/상품 단위 정밀 예측 · 자동 배포 |
| Cost | 내부 `cost-sim` 템플릿의 저비용 모델 단가 예시 기준으로, 전망/보고 질의 1회(`입력 10,000 + 출력 1,500 tokens` 가정)는 약 `$0.0024` 수준으로 본다. 월 100/500/1,000회 사용 시 약 `$0.24 / $1.20 / $2.40` 수준으로 추정하며, 첫 버전에서는 토큰 비용보다 데이터 정리와 운영 검토 공수가 더 큰 비용 변수일 가능성이 높다 |
